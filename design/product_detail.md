### 关键词权重提取规则

```
权重计算规则：
1. 词频权重：关键词出现次数 × 0.4
2. 位置权重：出现在"必须条件"中+0.5，"加分项"+0.3，"职责描述"+0.2
3. 语义强度："必须/must"×1.0，"优先/preferred"×0.6，"了解/familiar"×0.4
4. 综合权重 = 词频×0.4 + 位置×0.4 + 语义×0.2，归一化0-1

关键词等级分类：
- Critical（权重≥0.75）：必须具备
- Important（0.5-0.75）：重要加分
- Optional（<0.5）：有则更好
```

## 三、匹配算法逻辑

### 3.1 总体评分公式

```
总分（0-100）= 技术技能（0-35）+ 工作经验（0-25）+ 教育背景（0-15）
             + 软技能（0-15）+ 关键词覆盖（0-10）
```

### 3.2 各维度评分逻辑

**技术技能评分（满分35分）**

| JD要求等级 | 用户Expert | 用户Proficient | 用户Familiar | 用户无此技能 |
|-----------|-----------|---------------|-------------|------------|
| Expert | 100% | 70% | 30% | 0% |
| Proficient | 100% | 100% | 60% | 0% |
| Familiar | 100% | 100% | 100% | 0% |

- 必需技能满分25分，按权重计算
- 加分技能满分10分，按权重计算

**工作经验评分（满分25分）**

| 子维度 | 满分 | 评分规则 |
|-------|------|---------|
| 工作年限匹配 | 8分 | 满足要求=8分；差1年=-2分；差2年以上=0分 |
| 行业相关性 | 8分 | 同行业=8分；相近行业=5分；跨行业=1分 |
| 岗位相关性 | 9分 | 相同职能=9分；相近职能=6分；不相关=0分 |

**教育背景评分（满分15分）**

| 子维度 | 满分 | 评分规则 |
|-------|------|---------|
| 学历层次 | 8分 | 超出或满足要求=8分；低一级=4分；低两级=0分 |
| 专业相关性 | 7分 | 完全对口=7分；相关专业=4分；不相关=1分 |

**软技能评分（满分15分）**

- 从用户工作经历描述中NLP提取隐含软技能
- 从自我介绍中提取显性软技能声明
- 与JD要求软技能进行语义匹配（非精确关键词）
- 内置软技能映射数据库（如"项目管理"映射词：["跨团队协调","SCRUM","PMP","负责X项目"]）

**关键词覆盖评分（满分10分）**

- 提取JD Top20关键词
- 在用户简历全文搜索（精确+近义词语义匹配）
- 评分 = 覆盖率 × 10

### 3.3 投递建议决策树

```
综合评分 >= 70分  →  强烈推荐投递
综合评分 50-69分  →  优化后再投递（列出Top3改进项）
综合评分 < 50分   →  暂不建议投递（提供替代建议）

特殊规则：
- 存在Critical技能完全缺失 → 强制降级建议
- 学历硬条件不满足（如要求硕士，用户本科）→ 在建议中特别标注风险
- 工作年限差距超过3年 → 在建议中特别标注
```

## 四、简历生成策略

### 4.1 内容优化规则

**规则1：STAR法则量化改写**

```
原始：负责公司官网的运营工作
优化：主导公司官网SEO优化与内容运营，通过系统性关键词布局和内容更新策略，
     使网站月独立访客量从3万提升至12万（增长300%），有机流量占比提升至65%

改写规则：
1. 识别弱动词 → 替换强动词（负责→主导、参与→深度协作、做→设计/构建）
2. 检测是否有量化数据 → 若无，AI询问用户提供，或生成合理模板供确认
3. 每条成果描述控制在1-3行，不超过100字
4. 结合STAR结构：情境+任务+行动+结果
```

**规则2：动词升维对照表**

| 弱动词 | 升维动词 |
|--------|---------|
| 负责 | 主导、统筹、推动、把控 |
| 参与 | 深度参与、协同推动 |
| 做/完成 | 设计、构建、开发、交付 |
| 管理 | 带领、赋能、培养、统筹 |
| 提升 | 驱动提升、显著优化、突破性改善 |
| 帮助 | 赋能、支撑、推动 |

**规则3：ATS关键词植入策略**

```
植入三原则：
1. 自然融合：关键词出现在真实经历描述中，不简单堆砌
2. 密度控制：关键词出现频率控制在2-4次
3. 位置优先：Critical关键词优先出现在摘要/第一段工作经历中

植入方法：
- 方法A：在已有描述中替换近义词为JD使用的精确词汇
- 方法B：在工作职责末尾添加"所用技术/工具"句式
- 方法C：技能栏确保Critical关键词100%覆盖
- 方法D：个人简介中植入3-5个最重要岗位关键词
```

**规则4：简历结构排序策略**

| 用户类型 | 推荐结构 |
|---------|---------|
| 应届生 | 教育背景 → 实习经历 → 项目经历 → 技能 → 证书 |
| 社招（同行） | 个人简介 → 工作经历 → 技能 → 教育背景 → 证书 |
| 转行 | 个人简介（转行优势）→ 技能 → 相关项目 → 工作经历 → 教育背景 |
| 高管 | 核心成就 → 工作经历 → 教育背景 → 证书 |

## 五、异常场景处理

### 5.1 信息采集异常

| 异常场景 | 触发条件 | 处理策略 |
|---------|---------|---------|
| 用户回答模糊/过短 | 关键字段少于5个字 | AI追问："能详细描述一下吗？" |
| 用户回答跑题 | NLP识别与问题不匹配 | "我可能没理解对，你是想说...？" |
| 用户连续跳过 | 连续跳过3个问题 | 提示可随时返回补充，进入下一模块 |
| 数字数据缺失 | 工作成果无量化数据 | "能估算一个大概数字吗？如用户数、增长率" |
| 时间段重叠 | 两段工作时间有重叠 | 提示用户确认是否为兼职/同期情况 |
| 文件解析失败 | PDF加密/扫描版/损坏 | 告知原因，引导切换对话式采集 |
| 解析内容不足 | 解析出内容不足3个模块 | 提示结果可能不完整，引导补充 |

### 5.2 JD解析异常

| 异常场景 | 触发条件 | 处理策略 |
|---------|---------|---------|
| 链接无法访问 | 爬虫返回403/404/超时 | 告知链接失效，引导手动粘贴文本 |
| JD内容过短 | 提取文本<200字 | 提示JD内容较少，匹配精度可能受限 |
| JD非中文 | 识别为英文/其他语言 | 自动切换对应语言解析模式 |
| 关键字段缺失 | 无法提取经验年限/学历等 | 标记"未明确说明"，匹配时给予最低要求处理 |
| 重复JD | 用户提交相同JD | 提示"你已分析过此岗位，查看历史记录？" |
| 内容非招聘信息 | 识别为非JD文本 | 提示"检测到内容可能不是职位描述，请确认" |

### 5.3 匹配计算异常

| 异常场景 | 触发条件 | 处理策略 |
|---------|---------|---------|
| 档案信息不足 | 必填字段完成度<60% | 提示"你的档案信息还不够完整，匹配结果可能不准确，建议补充后再分析" |
| AI服务超时 | 匹配计算超过15秒 | 展示进度条，超时后提示重试；后台自动重试2次 |
| 评分极端情况 | 总分=0或=100 | 触发人工复核标记，日志记录，异常分析 |
| 技能识别失败 | 某技能在数据库中不存在 | 标记为"未知技能"，不计入评分，在报告中提示用户确认 |

### 5.4 简历生成异常

| 异常场景 | 触发条件 | 处理策略 |
|---------|---------|---------|
| 生成内容失真 | AI生成的成果描述与原信息严重偏差 | 内容比对检测，偏差度>40%时标记人工审核 |
| 关键词植入过密 | 某关键词出现超过5次 | 自动降低频率，分散到不同模块 |
| 内容过长 | 单页简历超过1.5页 | 自动压缩优先级低的内容，或提示用户精简 |
| 敏感信息检测 | 内容包含前公司保密信息 | AI识别并提示："检测到可能涉及保密信息，建议替换为通用描述" |
| 导出失败 | PDF/Word生成失败 | 展示纯文本版本，提示用户稍后重试 |

### 5.5 降级策略总览

```
服务降级策略（当AI服务不可用时）：
1. JD解析降级：仅做关键词提取（TF-IDF），不做语义理解
2. 匹配评分降级：仅计算关键词覆盖率，取消语义匹配
3. 简历生成降级：提供模板填充，不做AI内容优化
4. 对话采集降级：切换为表单填写模式
```
